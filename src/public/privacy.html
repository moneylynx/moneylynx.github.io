Politika privatnosti

Moja Lova by MoneyLynx

Posljednje ažuriranje: svibanj 2026. · Verzija 1.0

Ova Politika privatnosti opisuje kako aplikacija Moja Lova prikuplja, koristi i štiti podatke korisnika. Korištenjem aplikacije prihvaćaš ovu politiku.

1. Podaci koji se prikupljaju

Bez korisničkog računa (lokalni način rada)

Nikakvi podaci se ne šalju izvan tvog uređaja. Sve transakcije, postavke i preferencije pohranjuju se isključivo u localStorage preglednika ili internoj pohrani uređaja. Aplikacija može se koristiti u potpunosti offline.

S korisničkim računom (cloud sinkronizacija)

Prijavom putem Google računa ili e-mail/lozinke, sljedeći podaci se pohranjuju na sigurne Supabase poslužitelje:

  • E-mail adresa (iz Google profila ili ručnog unosa)
  • Ime i prezime (iz Google profila ili ručnog unosa)
  • Transakcije koje unosiš (iznosi, opisi, kategorije, datumi, statusi)
  • Postavke aplikacije (tema, jezik, valuta, vremenska zona)
  • Liste kategorija, lokacija i načina plaćanja koje kreiraš

Svaki korisnik može pristupiti isključivo vlastitim podacima. Pristup je zaštićen Supabase Row Level Security (RLS) pravilima na razini baze podataka.

2. Google Drive Backup (opcionalno)

Kada koristiš Google Drive backup funkcionalnost:

  • Eksplicitno autoriziraš pristup putem Google OAuth prozora
  • Aplikacija pohranjuje jedan backup JSON fajl u tvoj privatni appdata folder na Google Driveu
  • Tražimo isključivo opseg drive.appdata — aplikacija ne može čitati, mijenjati niti brisati bilo koji drugi fajl na tvom Google Driveu
  • Tvoje Google vjerodajnice se nikad ne pohranjuju na naše poslužitelje
  • Pristup možeš opozvati u bilo koje doba na: myaccount.google.com/permissions

3. Autentikacija

Google prijava: Koristimo OAuth 2.0 s PKCE protokolom putem Supabase Auth. Tvoja Google lozinka se nikada ne dijeli s ovom aplikacijom.

E-mail/lozinka: Lozinke hashira Supabase koristeći bcrypt algoritam. Aplikacija nikada ne vidi izvornu lozinku.

4. Enkripcija lokalnih podataka

Kada je PIN zaštita uključena:

  • Lokalni podaci enkriptirani su algoritmom AES-256-GCM
  • Ključ se derivira iz PIN-a koristeći PBKDF2 s 100.000 iteracija i SHA-256
  • PIN hash i salt pohranjuju se isključivo lokalno — nikada se ne prenose na server

5. Kolačići i lokalna pohrana

Web verzija aplikacije ne koristi kolačiće za praćenje, oglašavanje niti analitiku.

Aplikacija koristi:

  • localStorage — podaci aplikacije i korisničke preferencije
  • sessionStorage — privremeni ključevi sesijske enkripcije
  • Supabase auth tokeni — u localStorage za trajnost sesije prijave

6. Usluge trećih strana

  • Supabase (supabase.com) — baza podataka, autentikacija, real-time sinkronizacija
  • GitHub Pages (github.com) — web hosting
  • Google (accounts.google.com) — OAuth prijava
  • Google Drive API (googleapis.com) — isključivo za opcionalni backup
  • Google Fonts (fonts.googleapis.com) — učitavanje fontova

Navedene usluge mogu prikupljati standardne pristupne zapise (IP adresa, vrsta preglednika). Pogledaj njihove vlastite politike privatnosti za detalje.

7. Tvoja prava (GDPR)

  • Pristup podacima: Izvezi sve podatke u bilo koje doba (Postavke → Backup → Izvezi)
  • Brisanje cloud podataka: Odjavi se i pošalji zahtjev na gornju e-mail adresu
  • Brisanje lokalnih podataka: Briši pohrane preglednika ili deinstaliraj aplikaciju
  • Pravo na prenosivost: Podaci se mogu izvesti u JSON i CSV formatu
  • Opoziv Google pristupa: myaccount.google.com/permissions

Privacy Policy

Moja Lova by MoneyLynx

Last updated: May 2026 · Version 1.0

This Privacy Policy describes how Moja Lova collects, uses, and protects user data. By using the app you accept this policy.

1. Data We Collect

Without an account (local-only mode)

No data leaves your device. All transactions, settings, and preferences are stored exclusively in browser localStorage or the device's internal storage. The app works fully offline.

With an account (cloud sync)

When signed in via Google or email/password, the following data is stored on secure Supabase servers:

  • Email address (from Google profile or manual entry)
  • Display name (from Google profile or manual entry)
  • Transactions you enter (amounts, descriptions, categories, dates, statuses)
  • App preferences (theme, language, currency, timezone)
  • Category, location, and payment method lists you create

Each user can access only their own data. Access is protected by Supabase Row Level Security (RLS) policies at the database level.

2. Google Drive Backup (Optional)

When you use the Google Drive backup feature:

  • You explicitly authorise access via a Google OAuth popup
  • The app stores one backup JSON file in your private appdata folder on Google Drive
  • We request only the drive.appdata scope — the app cannot read, modify, or delete any other file in your Google Drive
  • Your Google credentials are never stored on our servers
  • You can revoke access at any time at: myaccount.google.com/permissions

3. Authentication

Google Sign-In: We use OAuth 2.0 with PKCE via Supabase Auth. Your Google password is never shared with this app.

Email/password: Passwords are hashed by Supabase using bcrypt. The app never sees your raw password.

4. Local Data Encryption

When PIN protection is enabled:

  • Local data is encrypted with AES-256-GCM
  • The key is derived from the PIN using PBKDF2 with 100,000 iterations and SHA-256
  • The PIN hash and salt are stored locally only — never transmitted to any server

5. Cookies & Local Storage

The web version does not use tracking, advertising, or analytics cookies.

The app uses:

  • localStorage — app data and user preferences
  • sessionStorage — temporary session encryption keys
  • Supabase auth tokens — in localStorage for session persistence

6. Third-Party Services

  • Supabase (supabase.com) — database, auth, real-time sync
  • GitHub Pages (github.com) — web hosting
  • Google (accounts.google.com) — OAuth sign-in
  • Google Drive API (googleapis.com) — optional backup only
  • Google Fonts (fonts.googleapis.com) — font loading

These services may collect standard access logs (IP address, browser type). See their respective privacy policies for details.

7. Your Rights (GDPR)

  • Data access: Export all data at any time (Settings → Backup → Export)
  • Delete cloud data: Sign out and send a request to the email below
  • Delete local data: Clear browser storage or uninstall the app
  • Data portability: Data can be exported in JSON and CSV formats
  • Revoke Google access: myaccount.google.com/permissions
